请结合实例说明Spring Boot自动配置的实现原理，并解释@SpringBootApplication注解包含哪些子注解及其作用

sprringboot 的自动配置是基于条件注解与自动扫描机制进行的，其核心思想是通过预定义的条件规则（如类路径依赖，配置属性等） 在应用启动时动态加载所需的bean，减少手动配置

夹具体的实现是：@springbootApplication注解
@springbootApplication是一个复合注解，包含三个注解：

  • @EnableAutoConfigration
  • @SpringbootConfigration
  • @ComponentScan

@SpringbootConfigration 标识这是一个spring的配置类，等同于传统的xml文件，标明可以通过@bean来定义bean

@EnableAutoConfigration 开启自动装配的注解， 通过spring factories 机制加载 META-INF/spring.factories文件中的自动配置类（如：DataSourceAutoConfiguration），并根据类路径依赖和条件注解决定是启用这些配置

@ComponentScan 自动扫描并注册bean ，默认扫描主类（@SpringbootApplication的类）所在包及其子包下的组件（@Component，@Service等）

实例：

  1. 引入spring-boot-starter-web
  2. @ComponentScan 注解 扫描主类的包（如：com.example.demo）
  3. @EnableAutoConfigration 出发自动配置，加载spring.factories中的WebMvcAutoConfigration，检测到DispatcherServlet 存在后自动配置mvc核心组件
  4. 无需手动配置

我们也以得出一个技巧： 我们不知道application.yml能配置什么，可以到xxxAutoConfigration中查看属性
