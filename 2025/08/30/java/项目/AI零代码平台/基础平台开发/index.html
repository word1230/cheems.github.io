库表设计

用户登录流程：

  1. 创建初始会话， 服务器生成一个匿名的session，将其ID 发给前端
  2. 登录成功，更新会话信息，记录用户信息。同时生成一个set-Cookie的响应头，让前端保存sessionID
  3. 前端保存session ID
  4. 后续请求都会携带该Cookie
  5. 后端从请求头的cookie中拿到sessionId ，得到用户数据
