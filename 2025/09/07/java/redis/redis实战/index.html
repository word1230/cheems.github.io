概述

这篇主要讲 redis 在springboot 中如何使用

基础概念

首先这是要通过程序对数据库进行操作 ， 参考 mysql ，它有jdbc这个库 可以与数据库交互
那我们redis 也有这样的库 ： jedis 与Lettuce 两个。

但是对于mysql，我们直接使用jdbc 非常麻烦 ， 于是大家发明了mybatis ，他是对jdbc 的封装
那我们redis 也有 ： spring data redis 它封装了jedis 与lettuce ，提供出了一些便捷的工具，让我们像是操作spring bean 一样操作redis

spring -data -redis 提供了一系列便捷的功能：

  1. 模板方法： RedisTemplate 与 StringRedisTemplate
    1. 提供了对不同数据结构进行操作的方法
    2. 内置了序列化器 ，自动完成二进制到数据结构的转换
    3. 支持多种序列化的方式
  2. 还借鉴了jpa 的思想， 可以直接根据方法名生成对应的方法
  3. 发布订阅模式的支持
  4. 事务支持 - 打包执行

思考题

一、数据结构设计

  1. 如果要记录“最近7天每天的访问次数”，用什么 Redis 数据结构？怎么设计 key？

使用string ， 在key 中存入日期， value就是访问次数 设置过期时间为7天
那么要获取每天的， 直接get 即可
要获取七天总和， 在业务层获取7次，然后加和。

二、 性能优化

  1. 如果访问量极大（10万QPS），单 key 自增会不会成为性能瓶颈？如何优化？

会， 因为redis 是单线程模型，任意时刻只有一个incr命令被执行，10万qps就会排队

两种思路解决：

  • 业务层进行累加， 定时任务进行redis更新

当接受请求时： 先通过java的LongAdder 或者ConcurrentHashMap 进行累加
设置定时任务，比如每一秒 将累加的数据更新到redis中

  • 客户端分片

思路：
不再只有一个Key，而是创建多个计数器（比如100个甚至1000个）。每次访问时，随机选择一个计数器进行增加。最后统计时，将所有分片计数器的值相加得到总和。

具体实现：
写入：

  1. 在java 的业务层 生成一个分片key 例如： visitor: 2025-09-07:${shard_id}
  2. shard_id 是一个从 0 - n-1 的数 （n 是分片数）
  3. 每次请求随机选择一个分片id ，执行命令

读取：

  1. 使用mget一次性获取所有分片key 的值
  2. 在客户端进行加和 ，获取总访问量

三、业务安全

  1. 如何防止恶意用户刷访问次数？

思路： 使用限流器， 如果同一ip/用户 短时间内频繁访问，超过一定阈值，拒绝

实现：

方案一： 基础限流

  1. 对于未登录的用户： 基于ip进行限流
    使用字符串进行记录

  2. 对于登录的用户： 基于用户id 进行限流
