优点

好处：

  • 环境隔离
    • 有多种环境， 每个环境都有一个应用运行
    • 那每个环境的前缀url一定是不同的
    • 使用这个， 就可以在启动任何环境时，就直接连接上对应环境中的应用，做相应的工作
    • 也就是- 一套代码，多处部署。无需修改任何代码，只需在启动时指定 spring.profiles.active=dev 或 prod，应用程序就会自动连接到对应环境的地址。
      • 如果不隔离，就需要手动修改代码，然后才能进行部署
  • 还有易于维护，如果以后前缀要修改，直接修改这里
  • 保证一致性： 避免手滑打错字了

不同环境

为什么要有环境

  1. 你不能在开发时往生产的数据库中写数据
  2. 你不能让测试用你本地的代码跑
  3. 你不能把调试日志，配置开关等带到线上

总之，开发，测试，生产等的数据，配置，资源不能混在一起。

企业中的环境

不同环境 = 不同配置 + 不同数据 + 不同权限 + 不同用途
企业中一般会有四个核心的环境 ：

  1. 开发环境
  2. 测试环境
  3. 预发布环境
  4. 生产环境

开发环境

  1. 使用人： 全体开发
  2. 部署位置： 公司内网
  3. 数据库： 所有开发共享
  4. 用途： 开发进行调试 ， 多人联调
  5. 配置文件： application-dev.yml

测试环境

  1. 使用人： 测试人员（QA）
  2. 部署位置： 独立的测试服务器
  3. 数据库： 独立的测试数据库
  4. 用途：
    1. 测试进行功能测试， 回归测试，压力测试
    2. bug修复验证
    3. 产品验收
  5. 配置文件： application-test.yml

预发布环境

  1. 使用人： 测试，运维，产品
  2. 部署位置： 与生产同架构的服务器
  3. 数据库： 生产库的只读副本
  4. 用途：
    1. 进行部署脚本的测试，配置，性能等的测试
    2. 产品验收
  5. 配置文件： application-pre.yml

生产环境

  1. 使用人： 用户
  2. 部署位置：线上服务器集群
  3. 数据库： 真是生产库（有备份，容灾，主从等）
  4. 用途：
    1. 对外提供服务
    2. 产生真实的业务数据
  5. 配置文件： application-prod.yml
